# Feature Specification: 목록 뷰 필터 UI 개선 (모바일 최적화)

**Feature Branch**: `013-filter-ui-mobile`  
**Created**: 2026-04-14  
**Status**: Draft  
**GitHub Issue**: #34

## 배경

리서치 탭의 필터 영역(카테고리 9개 + 우선순위 5개 + 예약상태 4개 칩)이 항상 펼쳐진 상태로 화면 상단을 차지하고 있다. 모바일 기준으로 필터 영역만 최대 150px 이상을 사용하며, 실제 목록을 보기 전에 스크롤이 필요하다. 또한 정렬 버튼도 필터 영역 아래에 별도로 노출되어 시각적 복잡도를 높인다.

이 개선은 "다음 버전 UX 계획"의 전체 탭 툴바 구조로의 전환 첫 단계다:
- 전체 탭 상단 툴바: [검색] [필터 버튼] [정렬 버튼]
- 필터 패널: 모바일 바텀시트 / 데스크탑 드롭다운

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 필터 없이 목록에 빠르게 접근 (Priority: P1)

사용자가 리서치 탭을 열었을 때 필터가 기본적으로 숨겨져 있어, 화면 최상단부터 곧바로 목록이 보인다. 필터를 사용하지 않는 방문의 경우(대부분의 경우) 불필요한 UI 요소가 공간을 차지하지 않는다.

**Why this priority**: 필터를 사용하지 않는 경우에도 항상 필터 영역이 노출되는 것이 현재의 핵심 문제다. 이 스토리가 해결되면 즉각적인 UX 개선이 발생한다.

**Independent Test**: 리서치 탭을 열어 검색창 바로 아래에 목록이 시작되는지 확인한다. 필터 칩 영역이 보이지 않아야 한다.

**Acceptance Scenarios**:

1. **Given** 필터가 하나도 활성화되지 않은 상태로 리서치 탭을 열면, **When** 페이지가 로드되면, **Then** 검색창과 필터/정렬 버튼만 보이고 카테고리·우선순위·예약상태 칩은 보이지 않는다.
2. **Given** 리서치 탭이 열려 있고, **When** 사용자가 스크롤 없이 화면을 보면, **Then** 목록의 첫 번째 아이템이 화면 내에 보인다.

---

### User Story 2 - 필터 패널을 열고 조건을 설정한다 (Priority: P1)

사용자가 "필터" 버튼을 탭하면 모바일에서는 바텀시트, 데스크탑에서는 드롭다운으로 필터 패널이 열린다. 패널 안에서 카테고리·우선순위·예약상태를 선택하고 닫으면 목록에 적용된다.

**Why this priority**: 필터 기능 자체는 유지해야 한다. 숨기되 접근성을 유지하는 것이 이 스토리의 핵심이다.

**Independent Test**: 필터 버튼 탭 → 패널에서 "식당" 카테고리 선택 → 패널 닫기 → 목록이 식당만 보이는지 확인.

**Acceptance Scenarios**:

1. **Given** 리서치 탭이 열려 있고, **When** "필터" 버튼을 탭하면, **Then** 모바일에서 바텀시트가 하단에서 올라오고, 데스크탑에서 드롭다운이 필터 버튼 아래에 열린다.
2. **Given** 필터 패널이 열려 있고, **When** 카테고리 "식당"을 선택하면, **Then** 선택 상태가 즉시 하이라이트되고 패널을 닫으면 목록에 적용된다.
3. **Given** 필터 패널이 열려 있고, **When** 바깥 영역을 탭하거나 "적용" 버튼을 누르면, **Then** 패널이 닫히고 선택한 필터가 유지된다.
4. **Given** 모바일 바텀시트가 열려 있고, **When** 아래로 스와이프하면, **Then** 패널이 닫힌다.

---

### User Story 3 - 활성 필터 상태를 한눈에 확인하고 해제한다 (Priority: P2)

필터가 하나 이상 활성화된 경우, 필터 버튼에 활성 개수 배지가 표시되고 검색창 아래에 요약 칩이 1줄로 표시된다. 요약 칩에는 X 버튼이 있어 개별 필터를 바로 해제할 수 있다.

**Why this priority**: 어떤 필터가 적용 중인지 파악하고 빠르게 해제하는 것은 필터 숨김 이후 필수 보완 기능이다.

**Independent Test**: 카테고리 "식당", 우선순위 "확정" 선택 → 버튼에 "(2)" 배지 확인 → 목록 위에 "식당 ×", "확정 ×" 칩 표시 확인 → "식당 ×" 탭 후 칩 제거 및 목록 갱신 확인.

**Acceptance Scenarios**:

1. **Given** 카테고리 2개, 우선순위 1개를 선택한 상태에서, **When** 필터 패널을 닫으면, **Then** 필터 버튼에 "(3)" 배지가 표시된다.
2. **Given** 활성 필터가 있는 상태에서, **When** 목록 상단을 보면, **Then** 각 필터 조건을 나타내는 요약 칩이 가로 스크롤 가능한 1줄에 표시된다.
3. **Given** 요약 칩이 표시된 상태에서, **When** 특정 칩의 X 버튼을 탭하면, **Then** 해당 필터 조건만 즉시 해제되고 목록이 갱신된다.
4. **Given** 활성 필터가 모두 없는 상태가 되면, **When** 요약 칩 영역을 보면, **Then** 요약 칩 영역이 사라진다.

---

### User Story 4 - 정렬 버튼으로 목록 순서를 변경한다 (Priority: P2)

현재 필터 영역 아래에 함께 있는 정렬 옵션이 별도의 "정렬" 버튼으로 분리된다. 버튼 탭 시 정렬 옵션(이름, 날짜, 예산, 우선순위)이 드롭다운 또는 메뉴로 표시된다. 현재 정렬 기준이 버튼 레이블에 표시된다.

**Why this priority**: 정렬은 필터와 별개의 기능이며, 분리하면 각각의 접근성이 높아진다.

**Independent Test**: "정렬" 버튼 탭 → "날짜" 선택 → 버튼 레이블이 "날짜 ↑"로 변경 → 목록이 날짜 오름차순으로 정렬되는지 확인.

**Acceptance Scenarios**:

1. **Given** 기본 정렬(이름 오름차순) 상태에서, **When** 정렬 버튼을 탭하면, **Then** 이름/날짜/예산/우선순위 옵션이 표시된다.
2. **Given** 정렬 옵션 중 "날짜"를 선택하면, **When** 선택하면, **Then** 정렬 버튼 레이블이 "날짜 ↑"로 바뀌고 목록이 날짜 오름차순으로 정렬된다.
3. **Given** "날짜 ↑" 정렬 상태에서, **When** 정렬 버튼을 다시 탭해 "날짜"를 선택하면, **Then** "날짜 ↓"로 토글되어 내림차순으로 정렬된다.

---

### Edge Cases

- 필터 조건을 모두 해제한 상태에서 "전체 초기화" 버튼이 노출되지 않아야 한다.
- 바텀시트가 열린 상태에서 다른 탭(지도, 테이블)으로 전환하면 바텀시트가 닫혀야 한다.
- 활성 필터 요약 칩이 많아서 1줄을 초과할 경우 가로 스크롤로 처리한다 (줄 넘김 없음).
- 필터 패널이 열린 상태에서 기기 회전(세로→가로)이 발생하면 바텀시트/드롭다운이 자연스럽게 유지되거나 닫혀야 한다.
- 검색어 입력 상태에서 필터를 추가로 적용해도 두 조건이 AND로 동시에 동작해야 한다 (현재 동작 유지).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 리서치 탭 최초 진입 시 카테고리·우선순위·예약상태 필터 칩이 화면에 보이지 않아야 한다.
- **FR-002**: 툴바에 "필터" 버튼과 "정렬" 버튼이 나란히 표시되어야 한다.
- **FR-003**: "필터" 버튼 탭 시, 768px 미만 화면에서는 바텀시트가 열려야 한다.
- **FR-004**: "필터" 버튼 탭 시, 768px 이상 화면에서는 버튼 아래 드롭다운 패널이 열려야 한다.
- **FR-005**: 필터 패널 안에 카테고리·우선순위·예약상태 선택 UI가 모두 포함되어야 한다.
- **FR-006**: 활성 필터가 1개 이상일 때 "필터" 버튼에 활성 개수 배지(숫자)가 표시되어야 한다.
- **FR-007**: 활성 필터가 1개 이상일 때 검색창 아래에 요약 칩이 가로 스크롤 1줄로 표시되어야 한다.
- **FR-008**: 각 요약 칩에는 X 버튼이 있어 해당 필터 조건만 개별 해제할 수 있어야 한다.
- **FR-009**: "정렬" 버튼 탭 시 정렬 옵션(이름/날짜/예산/우선순위)이 표시되어야 한다.
- **FR-010**: 현재 정렬 기준과 방향이 "정렬" 버튼 레이블에 표시되어야 한다 (예: "날짜 ↑").
- **FR-011**: 기존 필터 로직(AND 조합, 제외 항목 토글, 검색어 병행)은 변경 없이 유지되어야 한다.
- **FR-012**: 바텀시트는 아래로 스와이프하거나 외부 영역을 탭해 닫을 수 있어야 한다.

### Key Entities

- **FilterState**: 카테고리, 우선순위, 예약상태, 제외 표시 여부, 검색어 — 상태 자체는 변경 없음, 표현 방식만 변경
- **SortState**: 정렬 기준(key), 정렬 방향(asc/desc) — 상태 자체는 변경 없음, 표현 방식만 변경

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 리서치 탭 진입 후 목록 첫 번째 아이템이 스크롤 없이 화면에 보인다 (모바일 375px 기준).
- **SC-002**: 필터 조건을 선택하고 패널을 닫기까지 3탭 이하로 완료된다.
- **SC-003**: 활성 필터가 있는 경우 어떤 필터인지 목록 화면을 벗어나지 않고 확인할 수 있다.
- **SC-004**: 필터 초기화(전체 해제)가 목록 화면을 벗어나지 않고 가능하다.

## 가정 (Assumptions)

- 필터 상태는 페이지 새로고침 시 초기화되는 현행 방식을 유지한다 (URL param 직렬화는 이 이슈 범위 밖).
- "제외 항목 보기" 토글은 필터 패널 안으로 이동한다.
- 바텀시트 드래그 핸들 및 기본 애니메이션은 기존 ItemPanel의 바텀시트와 동일한 패턴을 사용한다.
- 데스크탑 드롭다운은 클릭 바깥 영역(outside click)으로 닫힌다.
