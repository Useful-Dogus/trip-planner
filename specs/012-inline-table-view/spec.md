# 012 — 인라인 편집 테이블 뷰

## 배경

현재 앱에서 일정 항목을 편집하려면 `항목 클릭 → 패널 열기 → 편집 모드 전환 → 필드 수정 → 저장`이라는 5단계를 거쳐야 한다. 항목이 많아질수록 이 흐름은 반복 작업에서 큰 마찰로 느껴진다.

사용자는 날짜별 일정을 정리할 때 구글 스프레드시트로 이탈하는 경향이 있다. 스프레드시트의 핵심 강점은 기능이 많아서가 아니라 **보는 공간과 편집하는 공간이 동일**하기 때문이다: 클릭하는 순간 편집 상태, 키보드만으로 다음 셀 이동, 전체를 한눈에 보면서 수정 가능.

이 피처는 날짜별 테이블 뷰를 신규 탭으로 추가하고, 테이블 위에서 직접 인라인 편집이 가능하도록 한다.

## 목표

- 항목의 주요 필드(시간, 이름, 카테고리, 상태, 예산)를 별도 폼 이동 없이 테이블 위에서 직접 편집
- Tab / Enter 키보드 내비게이션으로 여러 항목을 빠르게 연속 편집
- 날짜 그룹 하단의 빈 행에서 바로 새 항목 추가
- 저장은 blur 시 자동, 낙관적 업데이트로 즉각 반영

## 범위

### 포함

- `/schedule` 신규 페이지 (네비게이션에 탭 추가)
- `ScheduleTable` 컴포넌트: 날짜 그룹 테이블
- `DateGroupHeader`: 날짜, 일별 합계 예산, 접기/펼치기, 추가 버튼
- `TableRow`: 인라인 편집 가능한 행
- 셀별 인라인 편집 (시간, 이름, 카테고리, 상태, 예산)
- Tab / Enter / Escape 키보드 내비게이션
- 새 항목 추가: 고스트 행 클릭 또는 Cmd+Enter
- 행 오른쪽의 `···` 버튼으로 기존 상세 패널 열기
- Blur 시 자동 저장 + 낙관적 업데이트 + 실패 시 revert toast
- 모바일 반응형 (컬럼 축소 레이아웃)

### 제외

- 드래그로 순서/날짜 이동 (후속 이슈로 분리)
- 행 다중 선택 및 벌크 액션 (후속 이슈로 분리)
- 날짜 그룹 수동 추가/삭제 UI

## 레이아웃

### 탭 구성

```
[리서치]  [일정]  [지도]
```

- 리서치: 기존 ItemList + ResearchMap (변경 없음)
- 일정: 신규 ScheduleTable
- 지도: 기존 ScheduleMap (변경 없음)

### 테이블 구조

```
┌──────────────────────────────────────────────────────────────┐
│  4월 20일 (일) · D+1          합계 ¥42,000   [접기 ∧] [+ 추가] │
├──┬──────┬───────────────────┬──────────┬────────┬────────────┤
│⠿ │ 시간 │ 이름              │ 카테고리 │ 상태   │ 예산       │
├──┼──────┼───────────────────┼──────────┼────────┼────────────┤
│  │10:30 │ 나리타 공항 도착  │ ✈️       │ ● 확정 │            │
│  │14:00 │ 아사쿠사 탐방     │ 🏛️      │ ● 확정 │            │
│  │19:00 │ 스시 다이         │ 🍽️      │ ○ 예약 │ ¥8,000     │
│  │      │ + 항목 추가…      │          │        │            │ ← 고스트 행
├──┴──────┴───────────────────┴──────────┴────────┴────────────┤
│  4월 21일 (월) · D+2          합계 ¥18,000   [펼치기 ∨] [+ 추가] │
│  (접힌 상태)                                                   │
...
```

## 컬럼 설계

### 인라인 편집 컬럼 (테이블 노출)

| 컬럼 | 기본 표시 | 편집 방식 | Tab 순서 |
|------|----------|----------|---------|
| 시간 | `10:30` / `--:--` | 텍스트 입력 (`HH:MM`) | 1 |
| 이름 | 텍스트 | 텍스트 입력 | 2 |
| 카테고리 | 이모지 | 이모지 그리드 팝오버 | 3 |
| 상태 | 컬러 도트 + 짧은 라벨 | 4개 옵션 팝오버 | 4 |
| 예산 | `¥8,000` / 빈칸 | 숫자 입력 | 5 |

### 패널 전용 필드 (테이블 미노출)

- 주소 / 위경도 (지오코딩 연동)
- 메모
- 링크 (복수)
- `trip_priority` (리서치 단계 전용, 일정 테이블에서는 불필요)

### 상태(reservation_status) 시각화

```
● 확정        → 초록 도트  (예약완료)
◐ 예약완료    → 파랑 도트  (예약완료)
○ 예약필요    → 주황 도트  (필요·미예약)
— 불필요      → 회색 도트  (불필요)
? 확인필요    → 노랑 도트  (확인 필요)
```

## 인터랙션 상세

### 셀 편집

```
1. 셀 클릭 → 해당 셀 편집 상태 전환 (인풋 표시, 테두리 강조)
   다른 셀은 읽기 상태 유지 (맥락 유지)

2. Tab        → 같은 행의 다음 편집 가능 셀
   Shift+Tab  → 이전 셀

3. Enter      → 편집 확정 + 아래 행 동일 컬럼으로 이동
   Escape     → 편집 취소 (변경사항 되돌리기)

4. 셀 바깥 클릭 → blur → 자동 저장
```

팝오버 계열(카테고리, 상태) 셀은 선택 즉시 닫히고 다음 셀로 포커스 이동.

### 키보드 내비게이션 (편집 중이 아닌 상태)

```
↑ / ↓        → 행 이동
Enter         → 현재 행의 이름 셀 편집 시작
N             → 현재 날짜 그룹 하단에 새 행 추가
```

### 새 항목 추가

```
방법 1: 고스트 행 "+ 항목 추가…" 클릭
         → 이름 셀 즉시 편집 상태
         → Tab으로 시간 → 카테고리 → 상태 → 예산 순 이동
         → 마지막 셀 Tab 또는 Enter → 저장 + 다음 고스트 행 자동 포커스

방법 2: 행 선택 후 Cmd+Enter (Mac) / Ctrl+Enter (Windows)
         → 선택 행 아래에 새 행 삽입, 이름 셀 편집 시작
```

새 항목 생성 시 최소 필드: 이름만 입력하면 저장 가능. 나머지는 빈값 허용.

### 상세 패널 접근

- 행 오른쪽 `···` 버튼 클릭 → 기존 ItemPanel 열림 (주소, 메모, 링크 편집)
- 이름 셀 길게 클릭 (400ms hold) → 패널 열림

### 날짜 그룹 헤더

```
4월 20일 (일) · D+1       합계 ¥42,000    [접기 ∧] [+ 추가]
```

- **합계**: 해당 날짜 항목들의 `budget` 합산
- **D+숫자**: 여행 첫날(`date`가 가장 이른 항목) 기준 경과일
- **접기/펼치기**: 날짜 그룹 전체 collapse
- **+ 추가**: 고스트 행 포커스 (키보드 `N`과 동일)

### 빈 날짜 그룹

날짜에 항목이 아직 없는 경우:

```
4월 22일 (화) · D+3

    + 첫 번째 항목을 추가하세요
```

클릭 시 고스트 행 생성, 이름 셀 포커스.

## 저장 전략

### 낙관적 업데이트 + Debounce 저장

```
셀 값 변경
  → 로컬 React state 즉시 반영 (UI 즉각 반응)
  → 500ms debounce 후 PATCH API 호출
      ├─ 성공 → SWR 캐시 revalidate (별도 UI 피드백 없음)
      └─ 실패 → 이전 값으로 revert + toast ("저장에 실패했습니다. 다시 시도해주세요.")
```

### 저장 인디케이터

저장 중 스피너 / "저장됨" 메시지 **표시하지 않음**.
성공이 기본값인 환경에서 인디케이터는 노이즈. 실패할 때만 사용자에게 알린다.

### 연속 편집 중 Tab/셀 이동

Tab으로 셀을 이동할 때 이전 셀의 debounce 타이머를 즉시 flush하지 않는다.
행을 벗어나는 시점(다른 행 클릭 또는 고스트 행 이동)에 pending 타이머를 flush.

## 모바일 적응

데스크톱 5컬럼을 모바일에서 2줄 레이아웃으로 축소:

```
┌──────────────────────────────────────────┐
│ ✈️  나리타 공항 도착          ● 확정     │
│     10:30                                │
│ 🍽️  스시 다이                ○ 예약필  │
│     19:00                    ¥8,000      │
└──────────────────────────────────────────┘
```

- 1줄: 카테고리 이모지 + 이름 + 상태 도트
- 2줄: 시간 + 예산 (회색 보조 텍스트)
- 탭 키 이동 대신 탭 후 다음 행 이름 셀로 이동
- `···` 버튼 → 기존 바텀 드로어 패널

## 비기능 요구사항

- 날짜 그룹화 및 정렬은 `useMemo`로 메모이제이션
- 500ms 이상 소요되는 항목 로딩에는 기존 `ItemCardSkeleton` 재활용
- 편집 중인 셀이 있을 때 페이지 이탈 시 저장 flush (beforeunload)
