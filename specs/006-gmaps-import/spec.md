# Feature Specification: 구글맵 장소 가져오기

**Feature Branch**: `006-gmaps-import`
**Created**: 2026-04-06
**Status**: Draft

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 구글맵 리스트 URL로 장소 목록 불러오기 (Priority: P1)

사용자가 구글맵에서 공개 리스트의 공유 URL을 복사해 앱에 붙여넣으면, 해당 리스트에 저장된 장소들이 화면에 나타난다.

**Why this priority**: 기능의 핵심 진입점. 이 단계가 없으면 나머지 기능이 동작하지 않는다.

**Independent Test**: URL 입력 → 장소 목록 표시까지만 구현해도 "불러오기"의 가치를 독립적으로 검증할 수 있다.

**Acceptance Scenarios**:

1. **Given** 유효한 구글맵 공개 리스트 short URL이 있을 때, **When** 사용자가 URL을 입력하고 "불러오기"를 누르면, **Then** 리스트의 장소 목록(이름, 주소, 카테고리)이 화면에 표시된다.
2. **Given** 비공개로 설정된 구글맵 리스트 URL을 입력했을 때, **When** "불러오기"를 누르면, **Then** "비공개 리스트는 지원하지 않습니다" 오류 메시지가 표시된다.
3. **Given** 유효하지 않은 URL(구글맵 리스트 URL이 아닌 경우)을 입력했을 때, **When** "불러오기"를 누르면, **Then** "올바른 구글맵 리스트 URL을 입력해주세요" 오류 메시지가 표시된다.

---

### User Story 2 - 신규/유사 장소 구분하여 선택적으로 추가하기 (Priority: P2)

불러온 장소 목록을 검토하면서, 이미 DB에 있는 장소(중복) 또는 이름이 비슷한 장소(유사)는 경고와 함께 기본 선택 해제 상태로 표시되고, 신규 장소는 기본 선택 상태로 표시된다. 사용자가 원하는 항목만 선택해 추가한다.

**Why this priority**: 단순 일괄 추가와 달리, 중복 방지 로직이 실질적인 가치를 제공한다.

**Independent Test**: 미리 준비된 장소 목록(신규 + 중복 혼합)을 검토 화면에 표시하고 선택적으로 추가하는 전체 흐름을 테스트할 수 있다.

**Acceptance Scenarios**:

1. **Given** 불러온 장소 중 DB에 동일한 `google_place_id`가 존재하는 장소가 있을 때, **When** 검토 화면이 나타나면, **Then** 해당 장소는 "이미 추가됨" 표시와 함께 선택 불가 상태로 표시된다.
2. **Given** 불러온 장소 중 DB에 이름이 유사한 장소가 있을 때, **When** 검토 화면이 나타나면, **Then** 해당 장소는 기본값 선택 해제, "유사 장소: [기존 장소명]" 경고와 함께 표시된다.
3. **Given** 불러온 장소가 DB에 없는 신규 장소일 때, **When** 검토 화면이 나타나면, **Then** 기본값 선택 상태로 표시된다.
4. **Given** 검토 화면에서 일부 항목을 선택한 뒤, **When** "추가" 버튼을 누르면, **Then** 선택된 항목만 DB에 추가되고 "N개 장소가 추가되었습니다" 메시지가 표시된다.

---

### User Story 3 - 완전 중복 장소 자동 제외 (Priority: P3)

`google_place_id`가 정확히 일치하는 장소는 선택지 자체를 비활성화하여 실수로 중복 추가되는 상황을 방지한다.

**Why this priority**: P2의 세부 사항이지만 중복 제어 정책의 명확성을 위해 별도 구분한다.

**Independent Test**: 동일 `google_place_id`를 가진 장소가 DB에 있을 때 해당 항목이 비활성화되는지 검토 화면에서 확인.

**Acceptance Scenarios**:

1. **Given** 불러온 장소의 `google_place_id`가 DB의 기존 항목과 동일할 때, **When** 검토 화면이 표시되면, **Then** 체크박스가 비활성화(disabled) 상태이며 "이미 추가됨" 레이블이 표시된다.
2. **Given** 위 상태에서, **When** 사용자가 비활성화된 항목을 클릭해도, **Then** 선택 상태가 변경되지 않는다.

---

### Edge Cases

- 구글맵 내부 API 응답 구조가 변경된 경우 → "장소 목록을 불러오지 못했습니다. 잠시 후 다시 시도해주세요" 오류 표시
- 리스트에 장소가 0개인 경우 → "리스트에 장소가 없습니다" 안내 표시
- 네트워크 오류 또는 타임아웃 → 명확한 오류 메시지 표시
- 불러온 장소 수가 많은 경우(50개 이상) → 전체 목록 스크롤 가능하게 표시
- 구글맵에서 카테고리 정보가 없는 장소 → "기타"로 기본 분류
- 장소의 좌표(lat/lng) 정보가 없는 경우 → 주소만 저장, 지도 표시 제외

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 `maps.app.goo.gl/...` 형식의 short URL을 입력받아 구글맵 리스트 ID를 추출해야 한다.
- **FR-002**: 시스템은 추출한 리스트 ID로 해당 리스트의 장소 목록(이름, 주소, 위도/경도, 카테고리, `google_place_id`)을 조회해야 한다.
- **FR-003**: 시스템은 비공개 리스트 접근 시 명확한 오류 메시지를 표시하고 진행을 중단해야 한다.
- **FR-004**: 시스템은 불러온 각 장소를 기존 DB 데이터와 비교하여 "완전 중복", "유사", "신규"로 분류해야 한다.
- **FR-005**: 완전 중복(`google_place_id` 일치) 장소는 선택 불가(disabled) 상태로 표시해야 한다.
- **FR-006**: 유사 장소(이름 fuzzy match)는 기본값 선택 해제 + 기존 장소명 경고 표시 상태여야 한다.
- **FR-007**: 신규 장소는 기본값 선택 상태여야 한다.
- **FR-008**: 사용자는 검토 화면에서 각 항목의 선택 여부를 개별적으로 조정할 수 있어야 한다.
- **FR-009**: "추가" 버튼 클릭 시 선택된 항목만 DB에 INSERT되어야 한다.
- **FR-010**: 추가 완료 후 추가된 장소 수를 사용자에게 알려야 한다.
- **FR-011**: 구글 카테고리는 앱의 9개 카테고리(교통, 숙소, 식당, 카페, 관광, 공연, 스포츠, 쇼핑, 기타) 중 하나로 매핑되어야 한다.
- **FR-012**: DB 스키마에 `google_place_id` 컬럼이 추가되어야 한다.
- **FR-013**: 구글맵 연동 페이지는 앱 내비게이션에서 접근 가능해야 한다.

### Key Entities

- **GooglePlace**: 구글맵 리스트에서 가져온 장소 데이터 (이름, 주소, 위도, 경도, `google_place_id`, 구글 카테고리)
- **ImportCandidate**: 검토 화면에서 표시되는 단위 (GooglePlace + 분류 상태 + 기존 유사 장소명 + 선택 여부)
- **TripItem (확장)**: 기존 여행 아이템에 `google_place_id` 컬럼 추가

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 구글맵 공개 리스트 URL을 입력한 뒤 장소 목록이 30초 이내에 화면에 표시된다.
- **SC-002**: 불러온 장소 중 DB와 완전 중복(`google_place_id` 일치)인 항목은 100% 자동 감지된다.
- **SC-003**: 불러온 장소 중 이름 유사도가 높은 항목은 검토 화면에서 경고와 함께 표시된다.
- **SC-004**: "추가" 버튼 클릭 시 사용자가 선택한 항목과 실제 DB에 삽입된 항목이 100% 일치한다.
- **SC-005**: 비공개 리스트, 잘못된 URL, 파싱 실패 등 오류 상황에서 사용자에게 명확한 피드백이 제공된다.

## Assumptions

- 구글맵 리스트가 "공개(Public)" 설정인 경우에만 서비스가 동작한다. 비공개 리스트는 지원 대상 외다.
- 구글맵 내부 비공식 API를 사용하므로 응답 구조가 변경될 수 있으며, 파싱 실패 시 오류로 처리한다.
- 이름 유사도 판단 기준(임계값)은 개발자가 적절히 조정할 수 있도록 설정 가능하게 구현한다.
- 장소 추가 시 `status`는 "검토중", `priority`는 null로 기본값 설정한다.
- 장소 추가는 단방향(구글맵 → 앱)이며, 실시간 동기화는 지원하지 않는다.

## Out of Scope

- 기존 장소 수정 또는 삭제
- 구글맵과의 실시간 동기화
- 체인점 자동 묶음 처리
- 비공개 구글맵 리스트 지원
