# Feature Specification: 패널 기반의 끊김 없는 편집 경험

**Feature Branch**: `003-panel-editing-ux`
**Created**: 2026-03-25
**Status**: Draft
**Input**: Issue #29 - UX 개선 - 패널 기반의 끊김 없는 편집 경험

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 패널 UI 인프라: 사이드 패널 및 바텀 시트 (Priority: P1)

사용자가 리스트에서 항목을 클릭하면, 데스크탑에서는 리스트 우측에 사이드 패널이 슬라이드하여 나타나고 모바일에서는 화면 하단에서 바텀 시트가 올라온다. 리스트는 패널이 열려 있어도 계속 보이며 스크롤 가능하다.

**Why this priority**: 전체 기능의 기반이 되는 UI 컨테이너. 이 없이는 나머지 기능 구현 불가.

**Independent Test**: 리스트에서 항목 클릭 시 데스크탑/모바일에서 각각 패널/시트가 열리고, 닫기 버튼 및 ESC/스와이프로 닫힘을 확인. 리스트는 패널 뒤로 계속 보임.

**Acceptance Scenarios**:

1. **Given** 연구 리스트 페이지(데스크탑), **When** 항목 카드 클릭, **Then** 리스트 우측에서 사이드 패널이 슬라이드 인 되어 항목 상세 정보가 표시됨
2. **Given** 연구 리스트 페이지(모바일), **When** 항목 카드 클릭, **Then** 화면 하단에서 바텀 시트가 올라오며 항목 상세 정보가 표시됨
3. **Given** 패널/시트가 열린 상태, **When** ESC 키 입력(데스크탑) 또는 아래로 스와이프(모바일), **Then** 패널/시트가 닫힘
4. **Given** 패널이 열린 상태, **When** 패널 외부 배경 영역 클릭, **Then** 패널/시트가 닫힘

---

### User Story 2 - 인라인 읽기/편집 통합 인터페이스 (Priority: P2)

패널 내에서 항목 상세 정보를 읽다가 특정 필드를 클릭하면 해당 필드만 즉시 편집 가능 상태로 전환된다. 저장 시 리스트의 해당 카드가 실시간으로 갱신된다. 별도 페이지 이동 없이 패널 안에서 모든 편집이 완료된다.

**Why this priority**: 핵심 UX 목표인 "끊김 없는 편집 경험". 화면 전환 없이 편집할 수 있어야 작업 효율 향상.

**Independent Test**: 패널에서 항목 이름, 상태, 우선순위 등 필드를 클릭해 인라인 편집 후 저장하면 패널이 닫히지 않고 리스트 카드가 갱신됨을 확인.

**Acceptance Scenarios**:

1. **Given** 패널에서 상세 정보 읽기 모드, **When** 편집 버튼 클릭, **Then** 전체 편집 폼으로 전환되며 모든 필드 수정 가능
2. **Given** 편집 모드에서 변경 후 저장, **When** 저장 버튼 클릭, **Then** 데이터가 저장되고 패널은 읽기 모드로 복귀, 리스트 카드가 최신 데이터로 갱신됨
3. **Given** 편집 모드 중, **When** 취소 버튼 클릭, **Then** 변경사항이 폐기되고 읽기 모드로 복귀
4. **Given** 패널에서 항목 삭제, **When** 삭제 확인, **Then** 패널이 닫히고 리스트에서 해당 항목이 제거됨

---

### User Story 3 - 활성 상태 표시 및 맥락 기반 애니메이션 (Priority: P3)

패널이 열려 있을 때 리스트에서 현재 선택된 항목이 시각적으로 강조(Active State)된다. 패널 열림/닫힘에는 부드러운 전환 애니메이션이 적용되어 정보가 항목에서 확장되는 연결감을 준다.

**Why this priority**: 기능적 완성도보다는 폴리시에 해당. P1/P2가 동작하는 상태에서 추가.

**Independent Test**: 항목 클릭 시 해당 카드에 강조 스타일이 적용되고, 패널/시트 등장 시 슬라이드 애니메이션이 재생됨을 확인.

**Acceptance Scenarios**:

1. **Given** 패널이 열린 상태, **When** 리스트를 보면, **Then** 현재 선택된 항목 카드에 Active State 스타일(하이라이트/테두리)이 표시됨
2. **Given** 다른 항목 클릭, **When** 패널이 이미 열린 상태에서 다른 카드 클릭, **Then** Active State가 새 항목으로 이동하고 패널 내용이 갱신됨
3. **Given** 데스크탑에서 패널 열기, **When** 항목 클릭, **Then** 사이드 패널이 우측에서 슬라이드 인 애니메이션으로 등장
4. **Given** 모바일에서 시트 열기, **When** 항목 클릭, **Then** 바텀 시트가 하단에서 위로 슬라이드 인 애니메이션으로 등장

---

### Edge Cases

- 패널이 열린 상태에서 필터/정렬 변경 시 선택된 항목이 필터 결과에서 사라지면 패널을 닫음
- 편집 저장 중 네트워크 오류 발생 시 오류 메시지를 패널 내에 표시하고 패널을 닫지 않음
- 모바일 키보드 등장 시 바텀 시트가 키보드 위로 올라와 입력 필드가 가려지지 않아야 함
- 매우 긴 메모나 링크 목록이 있는 항목의 경우 패널/시트 내에서 스크롤 가능해야 함

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 사용자가 리스트 항목을 클릭하면 전체 화면 이동 없이 패널/시트 내에서 항목 상세 정보를 표시해야 함
- **FR-002**: 데스크탑(768px 이상)에서는 사이드 패널, 모바일(768px 미만)에서는 바텀 시트를 표시해야 함
- **FR-003**: 패널/시트 내에서 항목의 모든 필드를 편집하고 저장할 수 있어야 함
- **FR-004**: 편집 저장 시 페이지 이동 없이 패널과 리스트 카드가 최신 데이터로 갱신되어야 함
- **FR-005**: 데스크탑에서 ESC 키로 패널을 닫을 수 있어야 함
- **FR-006**: 모바일에서 아래로 스와이프하여 바텀 시트를 닫을 수 있어야 함
- **FR-007**: 패널이 열린 상태에서 해당 리스트 항목 카드에 Active State 스타일이 표시되어야 함
- **FR-008**: 패널/시트 외부 배경 클릭 시 닫혀야 함
- **FR-009**: 패널이 열린 상태에서 리스트는 계속 보이고 스크롤/필터 가능해야 함
- **FR-010**: 패널/시트 열림/닫힘에 슬라이드 애니메이션이 적용되어야 함

### Key Entities

- **Selected Item**: 현재 패널에서 표시/편집 중인 TripItem. id로 식별되며 패널 상태와 연결됨
- **Panel Mode**: 읽기(view) 또는 편집(edit) 상태를 나타냄. 항목 클릭 시 기본 읽기 모드, 편집 버튼 클릭 시 편집 모드
- **Panel Context**: 어떤 항목이 선택되어 있는지, 패널이 열려 있는지를 리스트와 패널 컴포넌트 간에 공유하는 상태

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 리스트에서 항목 상세 정보 확인까지 1번의 클릭으로 완료 (기존 2번: 클릭 + 페이지 로드 대기)
- **SC-002**: 항목 편집 완료까지 화면 전환 0회 (기존: 리스트→상세→편집→상세 3회 전환)
- **SC-003**: 패널/시트 열림 애니메이션이 300ms 이내에 완료
- **SC-004**: 편집 저장 후 리스트 카드가 새로고침 없이 즉시 갱신됨

## Assumptions

- 기존 `/items/[id]` 및 `/items/[id]/edit` 서버 페이지는 직접 접근 가능한 URL로 유지 (북마크, 공유 링크 지원)
- 패널 상태는 URL에 반영하지 않음 (세션 내 임시 상태)
- 새 항목 생성(`/items/new`)은 이 기능 범위에 포함하지 않음
- 데이터 저장 API는 기존 PUT `/api/items/[id]` 그대로 활용
