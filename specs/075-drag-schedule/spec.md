# 075-drag-schedule — 드래그로 일정 이동

GitHub: #75

## 문제

ScheduleTable 에서 항목의 날짜를 바꾸려면 상세 패널을 열어 `date` 필드를 직접 수정해야 한다. 여러 항목의 날짜를 조정하는 일정 짜기 흐름에서 패널 열기/닫기 반복 비용이 크다.

## 사용자 시나리오 — MVP (본 PR)

1. 사용자가 schedule 페이지에서 한 항목 카드(데스크탑) 또는 모바일 카드를 잡는다.
2. 다른 날짜의 그룹 헤더 위로 드래그한다 — 드롭존이 하이라이트된다.
3. 손을 떼면 `onUpdateItem(id, { date: targetDate })` 가 호출되어 그 날짜로 이동.
4. "날짜 미정" 그룹도 동일 인터페이스로 드롭 가능 (date=null).

## 범위 밖 — 후속 이슈로 분리

- **같은 그룹 내 순서 변경**: 현재 정렬이 `time_start` 자동 결정이라 그룹 내 manual reorder 는 데이터 모델 변경 필요 (별도 `order` 필드). 본 PR 범위 외.
- 날짜 그룹 헤더 자체의 reorder.
- 키보드 접근성 (dnd-kit KeyboardSensor) — 본 MVP 는 포인터/터치 우선.

## 완료 조건

- 데스크탑: 행을 잡아 다른 날짜 그룹 헤더로 드롭 → date 변경
- 모바일: 카드를 long-press(250ms) 후 다른 날짜 헤더로 드롭 → date 변경
- 드래그 중 hover 한 드롭존이 시각적으로 강조됨
- 드래그 취소 (현재 위치 또는 invalid 영역 드롭) 시 변경 없음
- 기존 셀 인라인 편집·"항목 추가"·접기 등 기능 회귀 없음
