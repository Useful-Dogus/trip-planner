# Feature Specification: Trip Planner 다음 버전 UX 개편

**Feature Branch**: `014-nav-ux-overhaul`  
**Created**: 2026-04-14  
**Status**: Draft  
**Input**: User description: "Trip Planner 다음 버전 UX 개편 (#77): 탭 구조 개편(리서치→전체, 추가/지도연동 탭 제거, 지도 탭 독립), 기기별 뷰 자동결정(모바일=카드고정/데스크탑=테이블고정), FAB 추가, 테이블 overflow 버그 수정, 일정탭 미배정 버킷+오늘 자동스크롤, 임포트 완료 후 자동이동+하이라이트, URL param으로 패널 상태 복원"

---

## 배경

1인 사용자의 여행 준비 앱. 현재 네비게이션은 4개 탭(리서치, 일정, 추가, 지도연동)으로 구성되어 있으나, 탭에 목적지와 액션이 혼재하고 탭 간 연결이 없어 아이템의 상태 변화(후보 → 결정 → 일정 배정) 흐름이 끊긴다. 모바일에서 테이블 뷰는 overflow 버그로 사용 불가 상태다.

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 탭 구조 개편 (Priority: P1)

사용자가 앱을 열면 **전체 / 일정 / 지도** 3개의 목적지 탭만 보인다. 추가와 지도연동은 탭에서 제거되고 각각 FAB / 툴바 버튼으로 이동한다.

**Why this priority**: 네비게이션은 모든 뷰의 진입점이므로 구조 개편이 선행되어야 이후 변경사항이 의미 있다.

**Independent Test**: 앱을 열어 네비게이션 탭이 3개(전체, 일정, 지도)인지 확인하고, 각 탭이 해당 뷰로 이동하는지 검증.

**Acceptance Scenarios**:

1. **Given** 앱이 열린 상태, **When** 네비게이션을 확인하면, **Then** 탭은 전체 · 일정 · 지도 3개만 표시된다.
2. **Given** 네비게이션 탭 3개, **When** 전체 탭 클릭, **Then** 전체 아이템 목록 뷰로 이동한다.
3. **Given** 네비게이션 탭 3개, **When** 일정 탭 클릭, **Then** 날짜 기준 일정 뷰로 이동한다.
4. **Given** 네비게이션 탭 3개, **When** 지도 탭 클릭, **Then** 통합 지도 뷰로 이동한다.
5. **Given** 전체 탭 화면, **When** 가져오기 버튼(툴바)을 클릭, **Then** 구글맵 임포트 플로우가 열린다.

---

### User Story 2 - 기기별 뷰 자동결정 (Priority: P2)

모바일에서는 카드 뷰만, 데스크탑에서는 테이블 뷰만 사용한다. 뷰 전환 토글은 제거된다.

**Why this priority**: 현재 모바일에서 테이블 뷰는 overflow 버그로 사용 불가다. 명시적으로 기기별 뷰를 고정하면 버그를 근본 해결하고 혼란을 제거한다.

**Independent Test**: 모바일 크기(< 768px) 화면에서 전체 탭을 열어 카드 목록이 보이는지 확인. 데스크탑(≥ 768px)에서 테이블이 보이는지 확인.

**Acceptance Scenarios**:

1. **Given** 화면 너비 < 768px, **When** 전체 탭에 진입, **Then** 카드 뷰만 표시되고 뷰 전환 토글은 없다.
2. **Given** 화면 너비 ≥ 768px, **When** 전체 탭에 진입, **Then** 테이블 뷰만 표시되고 뷰 전환 토글은 없다.
3. **Given** 데스크탑 테이블 뷰, **When** 가로 스크롤이 필요한 경우, **Then** 테이블이 가로 스크롤되고 내용이 잘리지 않는다 (overflow-hidden 버그 수정).

---

### User Story 3 - 모바일 FAB 추가 (Priority: P2)

모바일 전체 탭 우하단에 FAB(+) 버튼이 표시된다. 탭하면 새 아이템 추가 플로우로 진입한다.

**Why this priority**: 추가 탭이 제거되므로 모바일에서 아이템을 추가할 경로가 필요하다.

**Independent Test**: 모바일에서 전체 탭을 열고 FAB 버튼이 보이는지 확인. 버튼을 탭해서 새 아이템 폼이 열리는지 확인.

**Acceptance Scenarios**:

1. **Given** 모바일 전체 탭, **When** 화면을 보면, **Then** 우하단에 FAB(+) 버튼이 표시된다.
2. **Given** FAB 버튼이 표시된 상태, **When** FAB를 탭하면, **Then** 새 아이템 추가 폼이 열린다.
3. **Given** FAB 버튼, **When** 스크롤 중에도, **Then** FAB는 화면 고정 위치에 유지된다.
4. **Given** 모바일, **When** 일정/지도 탭 진입, **Then** FAB는 표시되지 않는다 (전체 탭 전용).

---

### User Story 4 - 임포트 완료 후 자동 이동 + 하이라이트 (Priority: P3)

구글맵에서 아이템을 가져온 후, 자동으로 전체 탭으로 이동하고 신규 추가된 아이템이 잠깐 하이라이트된다.

**Why this priority**: 현재 임포트 후 사용자가 "어디에 추가됐는가"를 수동으로 탐색해야 한다. 자동 이동+하이라이트로 인지 비용을 제거한다.

**Independent Test**: 구글맵 URL로 아이템을 가져오면 전체 탭으로 이동하고, 새 아이템 행/카드에 노란 배경이 500ms 동안 표시된 후 사라진다.

**Acceptance Scenarios**:

1. **Given** 임포트 완료 상태, **When** 완료되면, **Then** 자동으로 전체 탭으로 이동한다.
2. **Given** 전체 탭으로 자동 이동된 상태, **When** 신규 아이템이 표시될 때, **Then** 신규 아이템에 노란 하이라이트 효과가 나타났다 사라진다 (약 500ms).
3. **Given** 임포트가 0개 결과, **When** 완료되면, **Then** 전체 탭으로 이동하되 하이라이트는 없다.

---

### User Story 5 - 일정 탭 미배정 아이템 버킷 (Priority: P3)

일정 탭 최상단에 날짜가 없는 아이템을 모아 보여주는 "미배정" 섹션이 추가된다. 접기/펼치기 가능.

**Why this priority**: 현재 날짜 없는 아이템이 일정 탭에서 보이지 않아 편성 흐름이 끊긴다.

**Independent Test**: 날짜가 없는 아이템이 있는 상태에서 일정 탭을 열면 최상단에 "미배정 N개" 섹션이 표시된다.

**Acceptance Scenarios**:

1. **Given** 날짜 없는 아이템이 존재할 때, **When** 일정 탭에 진입, **Then** 최상단에 "미배정 N개" 섹션이 표시된다.
2. **Given** 미배정 섹션이 표시된 상태, **When** 섹션 헤더를 클릭, **Then** 섹션이 접히거나 펼쳐진다.
3. **Given** 날짜 없는 아이템이 0개, **When** 일정 탭에 진입, **Then** 미배정 섹션이 표시되지 않는다.
4. **Given** 미배정 섹션 내 아이템, **When** 아이템을 클릭, **Then** 아이템 편집 패널이 열린다.

---

### User Story 6 - 일정 탭 오늘 날짜 자동 스크롤 (Priority: P3)

여행 기간(배정된 아이템의 첫 날짜 - 마지막 날짜) 중에 앱을 열면 일정 탭이 오늘 날짜 섹션으로 자동 스크롤된다.

**Why this priority**: 여행 중(실행기 단계)에 오늘 일정을 빠르게 확인하는 것이 핵심 유스케이스다.

**Independent Test**: 오늘 날짜에 배정된 아이템이 있는 상태에서 일정 탭을 열면 오늘 섹션이 화면에 보인다.

**Acceptance Scenarios**:

1. **Given** 오늘 날짜에 배정된 아이템이 1개 이상 존재, **When** 일정 탭을 열면, **Then** 오늘 날짜 섹션으로 자동 스크롤된다.
2. **Given** 오늘 섹션으로 스크롤된 상태, **When** 오늘 섹션 헤더를 보면, **Then** "오늘" 배지가 표시된다.
3. **Given** 오늘 날짜에 배정된 아이템이 0개, **When** 일정 탭을 열면, **Then** 자동 스크롤 없이 상단부터 표시된다.
4. **Given** 여행 기간 밖의 날짜(여행 전/후), **When** 일정 탭을 열면, **Then** 자동 스크롤 없이 상단부터 표시된다.

---

### User Story 7 - URL param으로 패널 상태 복원 (Priority: P4)

아이템 패널이 열려 있을 때 URL에 `?item=ID`가 반영된다. 탭을 전환하고 돌아와도 패널이 복원된다.

**Why this priority**: 아이템 평가 중 지도를 잠깐 확인하고 돌아오면 패널이 닫혀 있는 문제를 해결한다. 다른 스토리들에 비해 체감 임팩트는 적다.

**Independent Test**: 아이템 패널을 열고 다른 탭으로 이동한 뒤 돌아왔을 때 패널이 자동으로 다시 열린다.

**Acceptance Scenarios**:

1. **Given** 아이템 패널이 열린 상태, **When** URL을 보면, **Then** `?item={ID}` 파라미터가 포함되어 있다.
2. **Given** `?item={ID}` URL, **When** 다른 탭으로 이동 후 뒤로 가기(브라우저 히스토리), **Then** 해당 아이템의 패널이 자동으로 열린다.
3. **Given** `?item={ID}` URL, **When** 다른 탭에서 직접 해당 탭 클릭, **Then** 해당 아이템의 패널이 열린다.
4. **Given** 패널을 닫으면, **When** URL을 보면, **Then** `?item` 파라미터가 제거된다.

---

### Edge Cases

- 임포트 중에 화면을 이탈하면 임포트 완료 후 자동 이동이 동작하지 않는다.
- `?item=존재하지않는ID` URL로 접근 시 패널이 열리지 않고 `?item` 파라미터가 제거된다.
- 오늘 자동 스크롤 중에 미배정 섹션이 많은 경우 오늘 섹션이 화면 밖에 있을 수 있음 → 미배정 섹션을 접은 상태로 자동 스크롤 시작.
- 데스크탑 화면을 모바일 크기로 리사이즈할 때 뷰가 즉시 전환된다 (CSS 반응형).

---

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 네비게이션은 전체 · 일정 · 지도 3개의 탭만 표시해야 한다 (추가 탭, 지도연동 탭 제거).
- **FR-002**: 전체 탭은 기존 리서치 탭을 대체하며 모든 아이템(확정된 아이템 포함)을 표시해야 한다.
- **FR-003**: 모바일(< 768px) 전체 탭은 카드 뷰만 표시하고 뷰 전환 토글을 제공하지 않는다.
- **FR-004**: 데스크탑(≥ 768px) 전체 탭은 테이블 뷰만 표시하고 뷰 전환 토글을 제공하지 않는다.
- **FR-005**: 데스크탑 테이블 컨테이너는 가로 스크롤을 지원해야 하며 내용이 잘려서는 안 된다.
- **FR-006**: 모바일 전체 탭 우하단에 FAB(+) 버튼이 표시되어야 하며, 탭하면 새 아이템 추가 폼이 열린다.
- **FR-007**: FAB는 화면 고정 위치에 표시되고 safe-area 위에 배치되어야 한다.
- **FR-008**: 구글맵 임포트 완료 후 자동으로 전체 탭으로 이동해야 한다.
- **FR-009**: 임포트된 신규 아이템은 전체 탭 진입 후 노란 하이라이트 효과가 나타났다 사라진다.
- **FR-010**: 일정 탭 최상단에 날짜 미배정 아이템을 표시하는 섹션이 있어야 하며, 접기/펼치기가 가능하다.
- **FR-011**: 미배정 섹션은 아이템이 0개일 때 표시되지 않는다.
- **FR-012**: 오늘 날짜에 배정된 아이템이 1개 이상 존재하면 일정 탭 진입 시 오늘 섹션으로 자동 스크롤한다.
- **FR-013**: 오늘 날짜 섹션 헤더에 "오늘" 배지가 표시되어야 한다.
- **FR-014**: 아이템 패널이 열릴 때 URL에 `?item={ID}` 파라미터가 반영되어야 한다.
- **FR-015**: `?item={ID}` URL로 진입하면 해당 아이템의 패널이 자동으로 열린다.
- **FR-016**: 존재하지 않는 ID가 `?item`에 있으면 파라미터를 제거하고 패널은 열지 않는다.
- **FR-017**: 구글맵 가져오기 진입점은 전체 탭 툴바 버튼으로 이동한다.

### Key Entities

- **아이템(Item)**: 여행 후보지/일정의 기본 단위. 이름, 카테고리, 우선순위, 예약상태, 날짜 속성을 가진다.
- **탭 상태**: 현재 활성 탭 (전체 / 일정 / 지도).
- **패널 상태**: 현재 열린 아이템 ID. URL `?item=` 파라미터로 직렬화된다.

---

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 새 장소를 발견하고 임포트하여 일정에 넣는 핵심 워크플로우가 탭 전환 1회 이하로 완료된다 (현재 3회).
- **SC-002**: 모바일에서 테이블 overflow 버그가 발생하지 않는다 (모바일에서 테이블 뷰 미제공으로 근본 해결).
- **SC-003**: 여행 기간 중 일정 탭을 열었을 때 오늘 날짜 섹션이 추가 스크롤 없이 화면에 보인다.
- **SC-004**: 임포트 완료 후 신규 아이템을 찾기 위한 수동 탐색이 불필요하다 (자동 이동 + 하이라이트).
- **SC-005**: 일정 탭에서 날짜 미배정 아이템이 0개 숨겨지지 않고 상단 버킷에 표시된다.
- **SC-006**: 아이템 패널을 열고 지도 탭을 확인한 뒤 돌아왔을 때 패널이 복원된다.

---

## 범위 제외 (Out of Scope)

- 다중 여행 프로젝트 지원 (로드맵: 2026-07 이후 별도 이슈)
- 필터·스크롤 상태의 URL 직렬화 (탭 전환 시 패널 상태만 복원; 필터/스크롤은 미포함)
- 지도 탭 내 기존 ResearchMap/ScheduleMap 통합 (현 단계에서는 지도 탭을 ResearchMap 기반으로 표시하되 ScheduleMap 제거는 별도 이슈)
- 키보드 단축키, PWA 지원, 내보내기 기능

---

## 가정

- 현재 `/research` 경로를 `/` 또는 `/research` 유지, 전체 탭으로 표시한다. 기존 bookmarks 호환 유지.
- 데스크탑 사이드바 하단에 지도연동 보조 버튼 배치는 범위에 포함한다.
- 하이라이트 지속 시간은 약 1000ms(500ms fade-in + 500ms fade-out) 기준으로 구현한다.
- "여행 기간"은 배정된 아이템 중 가장 이른 날짜 ~ 가장 늦은 날짜로 계산한다.
