# Trip Planner — 제품 개요 & 경쟁 제품 비교

작성일: 2026-06-22

## 1. 제품 컨셉

**리서치(후보 조사) → 결정 → 실행을 하나의 항목 테이블에서 관리하는 협업 여행 플래너.**

가고 싶은 장소를 모으는 단계와 확정된 일정을 짜는 단계를 별도의 앱·스프레드시트로 나눠 관리하는 번거로움을 없앤다. 모든 장소를 하나의 `items` 테이블에 넣고 **상태(검토중 / 보류 / 대기 / 확정 / 탈락)** 로 구분하면, 같은 데이터가 자동으로 리서치 뷰와 일정 뷰로 분리되어 보인다. 맥북·아이폰 브라우저를 모두 지원하는 웹앱이며 동행자 초대를 통한 공동 편집을 전제로 설계됐다.

타겟은 소규모 여행팀(2~5인). 출발점은 개발자 본인과 배우자의 2인 여행이었다.

## 2. 핵심 기능

| 영역 | 구현 내용 |
|------|-----------|
| 항목 관리 | 이름·카테고리·상태·우선순위·예약상태·주소·날짜/시간·예산·링크·메모를 가진 단일 항목 CRUD. 주소 입력 시 자동 지오코딩 |
| 지도 | Leaflet + OpenStreetMap. 카테고리별 색상 핀, 클릭 팝업, 수동/자동 중심점 설정 |
| 리서치 뷰 | 전체 후보를 카드(모바일)/테이블(데스크탑)로. 카테고리·상태·우선순위·예약 필터, 정렬, fuse.js 다중필드 검색 |
| 일정 뷰 | 확정 + 날짜/시간 있는 항목을 날짜별 그룹·타임라인으로. 일자별 지도(번호 핀 + 경로) |
| Google Maps 임포트 | 구글 저장 장소를 일괄 임포트(장소명·주소·좌표 파싱, place_id 보관) |
| 멀티유저·권한 | Supabase Auth + RLS. Owner/Editor/Viewer 역할, 멤버 초대 |
| 공유 | 비로그인 읽기전용 공유 링크(`/share/{token}`), OG 메타태그 |
| 성능·모바일 | SWR + localStorage 캐시, 낙관적 업데이트, 바텀시트 편집 패널, 키보드 회피 |

**기술 스택:** Next.js 14(App Router) · React 18 · TypeScript 5 · Tailwind · Leaflet · Supabase(PostgreSQL + RLS) · SWR · fuse.js

**개발 중·계획:** 다중 여행 대시보드(#112), trip CRUD 보강 등 기본기 게이트(G-1~G-22), 지도 고도화(Mapbox/3D/등시선), 시각화 페이지, PWA 오프라인, 드래그 일정 변경.

## 5. 핵심 차이와 포지셔닝

**우리만의 차별점은 "상태 기반 단일 테이블" 모델이다.** 트리플·Wanderlog는 '저장 위시리스트'와 '일정'을 별 개념으로 다루지만, 우리는 같은 항목이 상태값만 바뀌며 리서치 뷰와 일정 뷰를 오간다. "갈까 말까 고민 중인 50곳"을 버리지 않고 관리하다가 확정만 일정으로 끌어올리는 흐름은, 후보를 많이 쌓아두고 추려가는 실제 여행 준비 패턴과 잘 맞는다.

**경쟁 제품 대비 명확한 약점도 분명하다.** 트리플과 Wanderlog가 제공하는 (1) 동선·일정 자동 생성, (2) 예약 메일 자동 임포트, (3) 가이드/추천 콘텐츠, (4) OTA 예약·결제가 우리에겐 없다. 트리플은 사용자가 장소만 고르면 체류일수에 맞춰 일정을 짜주고, Wanderlog는 Gmail 연동으로 예약을 자동 정리한다. 이 세 가지는 대중 제품의 "킬러 편의 기능"이라, 일반 사용자 대상으로 정면 경쟁하긴 어렵다.

**현실적 포지션은 "스프레드시트를 대체하는 협업 리서치 도구"다.** Wanderlog와 비교 기사에서도 "구글 시트로 계획하되 지도를 붙이고 싶은 사람"이 자주 언급되는데, 우리 제품이 정확히 그 수요를 겨냥한다. 대중 OTA·콘텐츠 경쟁(트리플·마이리얼트립)으로 가기보다, 후보 관리의 유연함과 데이터 소유권(자체 호스팅) 을 무기로 "꼼꼼하게 직접 짜는 사람들"의 협업 도구로 좁히는 편이 승산이 있다.

**우선순위가 높은 격차 좁히기 후보:** ① 동선/이동시간 표시는 이미 `DistanceSeparator`로 일부 있으니 경로 순서 최적화로 확장, ② 구글 임포트가 있으니 예약 메일 임포트로 확장, ③ 실시간 협업 완성(현재 개발 중).
