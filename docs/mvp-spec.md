# Trip Planner — MVP 스펙

## 개요

- 목적: 뉴욕 10일 여행 플래너.
- 사용자: 본인, 아내.
- 주 사용 기기: 맥북, 아이폰. 브라우저 기반.

## 기술 스택

|               |                                   |
| ------------- | --------------------------------- |
| 레포          | GitHub (프라이빗)                 |
| 프론트/백엔드 | Next.js (App Router + API Routes) |
| 스타일        | Tailwind CSS                      |
| 데이터        | JSON 파일 (레포에 포함)           |
| 지도          | Leaflet.js + OpenStreetMap        |
| 실행          | 로컬 서버 + ngrok                 |
| 인증          | ID/PW + JWT                       |

## 구조

항목 테이블 하나. 리서치와 일정을 분리하지 않음.

- 상태 "확정" + 날짜/시간 있음 → 일정 뷰에 표시
- 그 외 → 리서치 뷰에 표시

## 데이터 모델

```
id
name          // 이름
category      // 교통 | 숙소 | 식당 | 관광 | 쇼핑 | 기타
status        // 검토중 | 보류 | 대기중 | 확정 | 탈락
priority      // 반드시 | 들를만해 | 시간 남으면
address       // 주소
lat / lng     // 좌표 (주소 입력 시 자동 변환)
links[]       // { label, url } 복수
budget        // 예상 비용
memo          // 자유 텍스트 + 이미지
date          // 방문 날짜
time_start    // 시작 시간
created_at / updated_at
```

### 상태 / 우선순위

| status |                        | priority    |                       |
| ------ | ---------------------- | ----------- | --------------------- |
| 검토중 | 아직 결정 안 됨        | 반드시      | 이것 때문에 뉴욕 간다 |
| 보류   | 나중에 다시 볼 것      | 들를만해    | 근처 가면 꼭 들른다   |
| 대기중 | 예약 오픈 등 조건 있음 | 시간 남으면 | 여유 있으면 간다      |
| 확정   | 간다                   |             |                       |
| 탈락   | 안 간다                |             |                       |

> 목록에선 키워드 배지만. 셀렉터 열면 설명 함께 표시.

## 기능

### 인증

ID/PW 로그인, JWT 세션, 로그아웃. 회원가입 없음 (계정 하드코딩).

### 리서치 뷰 — 전체 항목

- 목록: 카테고리 / 상태 / 우선순위 필터
- 지도: 탈락 제외 전체 핀, 카테고리별 색상, 클릭 시 요약 팝업
- 항목 추가 / 수정 / 삭제

### 일정 뷰 — 확정 + 날짜 있는 항목만

- 목록: 날짜별 그룹, 시간순 정렬
- 지도: 날짜 선택 → 번호 핀 + 동선 연결
