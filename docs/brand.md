# Waypost 브랜드 가이드

> 브랜드 디스커버리(이슈 #231)의 산출물. 제품 정체성·톤·이름·로고·컬러 팔레트를 한곳에 정의한다.
> 색·이름·로고를 다루는 모든 작업(#228 색상 토큰, #229 네이밍 노출)은 이 문서의 결정을 기준선으로 한다.
> 시각 토큰 상세(타이포·스페이싱·섀도 등)는 [design-guidelines.md](design-guidelines.md) 를 따른다.

---

## 1. 제품 정체성 (한 문장)

> **"AI 가 짜주는 일정이 아니라, 지도 위에서 후보를 모아 동선을 직접 깎는 여행 설계 도구."**

핵심은 *내 손으로 만드는 여행 지도* 다. 자동 생성 일정의 반대편에 선다. 사용자는 지도 위에 후보 장소를 모으고, 일자별로 동선을 직접 배치·조정한다. 지도가 1순위 콘텐츠이며 UI 는 거기에 양보한다(= design-guidelines 의 Deference 원칙과 동일 방향).

## 2. 타겟 사용자 / 사용 맥락

- 직접 일정을 짜는 걸 즐기는 여행자 (개인 + 소규모 그룹).
- 후보지를 여러 곳 모아두고 동선·일자를 손으로 매만지며 계획을 숙성시키는 사람.
- "버튼 하나로 일정 자동 생성" 을 원하는 사용자가 1차 타겟이 **아니다**. Waypost 는 설계의 통제권을 사용자에게 둔다.

## 3. 톤 & 무드

**차분 · 정교 · 신뢰** — 카토그래퍼(지도 장인)의 작업대.

- 신뢰감/차분함을 우선한다. 과한 모험감·들뜸·네온은 피한다.
- 시각적 잡음 최소화. 강조는 드물게, 의미 있을 때만.
- "AI 가 빠르게 생성한 프로토타입" 인상을 정면으로 반박하는 것이 디스커버리의 출발점이었다. 따라서:
  - 채도 높은 보라/사이언/네온 그라데이션 **금지**.
  - 따뜻한 크림/세피아 종이 텍스처 바탕 **금지** (스큐어모픽 '지도 종이' 는 오히려 AI 데모 클리셰).
  - 차가운 중립 바탕 + 잉크 + 절제된 메탈릭 강조로 카토그래픽 정체성을 만든다.

## 4. 이름 — Waypost

- **Waypost** (working name). `way` + `post` = 이정표 / 길잡이(signpost). "길을 짚어가는 지표" 라는 의미가 지도공방 정체성과 직결된다.
- 한국어 발음: "웨이포스트". 한국어 UI 와 병기 자연스러움.
- **잠정 이름이며 추후 변경 가능하다.** 정식 출시 전 아이디어 숙성 단계의 working name 으로 채택했다. 이름 변경 시 워드마크 컴포넌트(#229) 한 곳만 바꾸면 표면 전체에 반영되도록 설계한다.

### 검토 기록

- **방향:** A 지도공방(Cartographer). B 설렘 동반자(소비자 톤, 경쟁 앱과 겹침)·C 차분 정리(현 teal 진화, 차별성 약함) 대비 "AI 생성 인상" 을 가장 강하게 반박.
- **이름 후보 충돌 검증:** Plott ❌(Plotline 여행앱 등 다수), Routely ❌(다수 선점), Mapwright/Trailmark ❌(선점), Wayfold·Pinwise(검색 무충돌이나 도메인은 최근 선점 정황).
- **도메인 현실:** 발음 가능한 짧은 이름은 코인드 포함 `.com` 사실상 전부 점유. Waypost 의 `.com` 은 개인 보유(제품 아님, 0xbf.com 리다이렉트)로 활성 경쟁 제품 없음. `waypost.kr` / `.me` / `.xyz` 가용 → 한국 시장 앱이므로 **waypost.kr** 을 기본 도메인으로 한다.
- 한국어발음→영어 변형(길잡이→Giljabi, 둘레→Dulle, 여정→Yeoju)도 검토했으나 의미 직관성·발음·가용성 종합에서 Waypost 우위.

## 5. 로고 / 워드마크 컨셉

- **워드마크:** "Waypost" 텍스트, 잉크 네이비, SemiBold. 자간 -0.01em.
- **마크(심볼):** 동선·이정표 모티프 — 두 웨이포인트(점)를 잇는 곡선 경로의 **전용 글리프**(`WaypostGlyph`, 정본 형상: `app/icon.svg`). 헤더·로그인·빈 상태에서는 이 글리프를 브라스 원형 테두리 안에 두고(`BrandMark`), favicon·앱 아이콘은 잉크 네이비 라운드 사각 배경에 같은 글리프를 둔다. **글리프 형상은 단일 소스(`components/Brand/Wordmark.tsx` 의 `WaypostGlyph`)로 통일**하며 lucide 아이콘을 마크 용도로 쓰지 않는다.
- 모노톤(잉크) 버전과 브라스 강조 버전 둘 다 운용. 다크 배경에서는 밝은 잉크(거의 흰색) + 밝은 브라스.
- 단독 마크는 정사각/원형, 워드마크는 마크 + 텍스트 가로 조합.

## 6. 컬러 팔레트

브랜드 색은 **쿨 그레이 바탕 + 잉크 네이비 + 브라스 강조** 3축. 라이트/다크를 각각 별도 설계한다(단순 반전 아님). 실제 토큰 적용은 #228 에서 `app/globals.css` + Tailwind theme 로 수행하며, 값은 아래를 정본으로 한다.

### 6.3 규칙

- **강조(브라스)는 절제.** CTA·활성 상태·동선/핀 등 의미 있는 곳에만. 채도 높은 주황(‘AI 스타트업’ 색) 직접 사용 금지.
- status 색(success/warning/critical/info)·카테고리 색 체계는 design-guidelines §3·§8 을 유지한다(브랜드 강조와 별개의 기능색).
- 다크모드는 명도 반전이 아닌 층 구조: Base(`--bg`) → Elevated(`--bg-elevated`) 로 갈수록 한 단계 밝게.
- 기존 teal accent(`13 148 136` / `45 212 191`)는 brass 로 전량 교체한다(#228).

## 7. 타이포 / 아이콘 톤

- 타이포 스택·스케일은 design-guidelines §2 를 그대로 따른다(Pretendard 우선). 워드마크만 SemiBold + 자간 -0.01em.
- 아이콘은 lucide-react 단일 패밀리(design-guidelines §8). 카토그래픽 톤에 맞춰 `route` · `map-pin` · `compass` · `signpost` 계열을 브랜드 모티프로 우선 활용.
- 강조 아이콘에만 브라스, 일반 아이콘은 `--fg` / `--fg-muted`.
